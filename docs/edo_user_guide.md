# ЭДО — пользовательский гайд

Внутренний документооборот: служебные записки, заявления, уведомления,
командировочные сметы. Документы создаются через типизированные формы,
проходят цепочку согласования и архивируются с PDF-копией.

## С чего начать

1. **Меню → ЭДО** — попадаешь в свой личный кабинет.
2. Четыре вкладки сверху:
   - **Ждут меня** — документы, по которым требуется твоё решение (с красной плашкой-счётчиком).
   - **Мои в работе** — что отправил на согласование, ещё не закрылось.
   - **Черновики** — заполненные, но не отправленные.
   - **Архив** — закрытые: согласованные, отклонённые, отменённые.

## Создать документ

1. Кнопка **Создать** в правом верхнем углу.
2. Каталог типов сгруппирован по категориям: Служебные записки / Заявления /
   Уведомления / Командировки / Премирование. Кликаешь нужный тип.
3. Заполняешь форму. Поля бывают:
   - текст, многострочный, число, сумма (₽), дата, период (от–до), время;
   - выбор из списка (choice), флажок (Да/Нет);
   - сотрудник / несколько сотрудников / организация / подразделение;
   - таблица (для списков премий — добавление/удаление строк по кнопкам);
   - файл.
4. Внизу страницы — справочно показана **цепочка согласования**: кто за кем
   будет смотреть документ, с указанием ролей и SLA.
5. Кнопки:
   - **Сохранить черновик** — оставит документ на вкладке «Черновики», номер
     не присваивается.
   - **Отправить на согласование** — присваивается номер, документ уходит
     первому в цепочке.

### Особые случаи

- **Премирование** (`memo_bonus_*`) — создаёт только руководитель
  подразделения. В таблицу добавляешь сотрудника + сумму + обоснование;
  итоговая сумма пересчитывается автоматически.
- **Командировка** (`travel_estimate`) — итог = транспорт + проживание +
  суточные (пересчитывается на лету). На последнем шаге согласования
  (директор) понадобится **рисованная подпись** — место для неё появится
  в панели одобрения.
- **Уведомление об отпуске** (`vacation_notification`) — создаёт **только
  бухгалтерия**. Бухгалтер выбирает сотрудника, документ подписывают
  по очереди: сначала бухгалтер, потом сотрудник, после уведомление
  идёт его руководителю «к сведению».

## Согласование документа

1. Открываешь документ из вкладки «Ждут меня» (или из ссылки в email).
2. Слева — тело документа и поля, справа — степпер цепочки и панель **Ваше решение**.
3. Можно:
   - **Согласовать** — комментарий опционален. Если документ требует
     рисованной подписи — подпиши пальцем/мышью на холсте.
   - **Запросить правки** — комментарий обязателен; документ возвращается
     автору в статус «Запрошены правки», тот правит и отправляет повторно.
   - **Отклонить** — комментарий обязателен; финальный статус, повторное
     согласование невозможно (нужно создавать новый документ).
   - **Делегировать** — выбираешь другого сотрудника, документ переходит к нему.

### Email-link согласование

В каждое уведомление о новом шаге вшиваются две ссылки — **Согласовать /
Отклонить**. Клик ведёт на публичную страницу, на которой можно принять
решение без логина (авторизация — по подписанному токену). Срок действия
ссылки — 14 дней. Для отклонения по ссылке всё равно нужен комментарий.

### Параллельные ветки

Если в цепочке несколько шагов помечены одним `parallel_group`, они
становятся активными одновременно — у тебя в «Ждут меня» появится
документ от нескольких параллельных согласующих.

- **Режим AND** — нужно одобрение всех в группе, любой `reject` сразу
  закрывает документ.
- **Режим OR** — достаточно одного `approve`, остальных автоматически
  снимет (статус «Пропущен»). При `reject` в OR-режиме документ закроется,
  только когда все в группе отклонят.

## Замещение на время отпуска

В **Профиле** можно настроить:
- замещающего (кто принимает решения вместо тебя);
- даты отсутствия.

Когда наступит этот период, любой новый шаг согласования, который должен
был активироваться у тебя, автоматически уйдёт замещающему. В истории
шага останется ссылка на тебя как `original_approver`.

## SLA и просрочки

Каждый шаг согласования имеет срок (SLA). Если шаг висит дольше срока,
система раз в час фиксирует просрочку и шлёт уведомления:
- автору (информационно);
- согласующему (с напоминанием);
- его непосредственному руководителю (для эскалации).

Просроченные шаги видны на отчёте «Нарушения SLA» (только админам).

## Связи между документами

На странице любого документа есть карточка **Связанные документы**.
Кнопкой **Связать** можно прикрепить другой документ ЭДО (служебка↔приказ,
смета↔отчёт). Поиск — по номеру или заголовку.

## PDF-экспорт

В правом верхнем углу страницы документа — кнопка **PDF**. Возвращает
PDF с шапкой/подвалом и подписями. Кеш — 7 дней, инвалидируется при любом
изменении полей или подписей.
